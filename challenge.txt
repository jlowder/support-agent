The Challenge: AI Customer Support Agent

Build a fully functional web application for an AI Customer Support
Agent that processes or denies e-commerce refunds using an LLM to
bootstrap your development. It needs:

1. Mock Data: A CRM database (15 profiles) and a strict refund
policy document.

2. Agent Backend: An agent loop (LangGraph, CrewAI, or raw function
calling) that dynamically calls tools to validate policy rules.
Bonus: Integrate a voice pipeline (e.g., OpenAI Realtime API,
ElevenLabs, or LiveKit).

3. Frontend UI: A clean interface featuring a customer chat interface
and/or a microphone voice component, alongside an admin dashboard
showing real time agent reasoning logs.

Main Deliverable: Loom Video Walkthrough

Your main evaluation will be a 7–10 minute Loom video. Please walk
us through:

● Live Demo: Show the agent handling a standard refund and an edge
case/policy violation ("holding the line"). If you implemented voice
features, demonstrate a live spoken interaction.

● Code Tour: Walk through your repository architecture, tool
orchestration, and any voice stream handling.

● Reasoning Logs: Show where the agent handles failures or retries
in your admin panel or terminal trace.  Note: Please include your
public GitHub repository link and a clean README along with your
Loom video.  Timeline & Next Steps

● Deadline: Please submit your Loom and GitHub links within 7
calendar days.

● Final Step: A successful review leads to a final 30-minute
conversational interview with the team. We are excited to see what
you build!
